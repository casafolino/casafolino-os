## Evidenza

### 1. Serper API key MANCANTE in produzione

```
Query: SELECT key, LENGTH(value) FROM ir_config_parameter WHERE key ILIKE '%serper%';
Risultato: 0 rows
```

La chiave `casafolino.serper_api_key` non esiste in `ir_config_parameter`. Il codice `action_enrich_007()` a riga 447 la rilegge con `get_param('casafolino.serper_api_key', '')` e ottiene stringa vuota. Il blocco Serper (righe 449-554) viene saltato perche` `if serper_key:` e` False.

**Conseguenza**: il prompt inviato a Groq contiene `"Nessun risultato trovato dalla ricerca web."` come search_context. Il LLM non ha dati reali su cui basarsi e restituisce solo valori null o generici.

### 2. Groq API key PRESENTE

```
casafolino.groq_api_key | 56 chars
```

La chiave Groq esiste e ha lunghezza plausibile (56 caratteri).

### 3. Solo 4 partner arricchiti su 16.178

```
total: 16178 | enriched: 4
```

I 4 partner arricchiti (id 98891, 100262, 100291, 15) hanno tutti:
- `cf_007_potenziale = 'basso'`
- `cf_007_fatturato = NULL`
- `cf_007_settore = NULL`
- `cf_007_enriched_from = 'email'`

Confermano che l'enrichment gira ma senza dati Serper il LLM restituisce quasi tutto null.

### 4. Nessun cron per enrichment automatico

```
Query: cron_name ILIKE '%enrich%' OR '%007%' OR '%agente%' OR '%intelligence%' OR '%rebuild%'
Risultato: solo "CRM: enrich leads (IAP)" (cron Odoo standard, non CasaFolino)
```

I file XML cron (`cf_mail_cron.xml`, `casafolino_mail_hub_cron.xml`) sono entrambi vuoti (`<odoo></odoo>`).

Il metodo `_rebuild_top_partners()` in `casafolino_partner_intelligence.py:701` calcola hotness/NBA, **non** lancia l'enrichment 007. E non ha nemmeno un cron registrato.

### 5. BUG: `action_enrich_from_domain()` non esiste

Il controller `mail_v3_controllers.py:636` chiama `partner.action_enrich_from_domain()` ma questo metodo **non esiste** da nessuna parte nel codebase. Qualsiasi click su "Enrich" nella sidebar 360 genererebbe un `AttributeError`.

### 6. Nessun log di enrichment nei log recenti

```
docker logs odoo-app --tail=2000 | grep -iE 'enrich|groq|serper|007|agente'
Risultato: nessuna riga relativa a enrichment
```

## Root Cause (multiplo)

| # | Causa | Impatto |
|---|---|---|
| **C1** | `casafolino.serper_api_key` non configurata in produzione | Il LLM non riceve dati reali dal web, restituisce campi null |
| **C2** | Nessun cron per enrichment automatico | L'enrichment funziona solo se un utente clicca manualmente il bottone |
| **C3** | `action_enrich_from_domain()` non implementata | Il bottone "Enrich" nella sidebar 360 genera errore (metodo mancante) |
| **C4** | Nessun cron per `_rebuild_top_partners()` | Hotness/NBA non viene mai calcolato automaticamente |

Lo scenario principale e` una combinazione di **(C1)** + **(C2)**:
- Anche quando un utente lancia 007 manualmente, senza Serper il LLM produce dati vuoti.
- Non esiste automazione, quindi i 16K partner non vengono mai toccati.

## Ipotesi di fix (NON implementate)

### Fix C1 -- Configurare Serper API key
```sql
INSERT INTO ir_config_parameter (key, value, create_uid, create_date, write_uid, write_date)
VALUES ('casafolino.serper_api_key', 'CHIAVE_SERPER_QUI', 1, NOW(), 1, NOW());
```

### Fix C2 -- Creare cron per enrichment automatico
Registrare un `ir.cron` in `cf_mail_cron.xml` che chiami un metodo `@api.model` su `res.partner` per arricchire batch di partner non ancora enriched (`cf_007_enriched = False`) con rate limiting (es. 50 partner/ora).

### Fix C3 -- Implementare `action_enrich_from_domain()`
Creare il metodo su `res.partner` in `cf_contact.py` che:
1. Estrae il dominio dall'email del partner
2. Cerca/crea un partner company con quel dominio
3. Linka il partner persona alla company (`parent_id`)
4. Opzionalmente lancia `action_enrich_007()` sulla company

### Fix C4 -- Creare cron per intelligence rebuild
Registrare un `ir.cron` che chiami `_rebuild_top_partners()` periodicamente (es. ogni 6 ore).
